# Plan 01-02: Custom Trailing Cursor and Scroll Progress Bar

## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Rendering lag of trailing ring | Medium | Use `gsap.quickTo` for spring-like translation coords updates to avoid frame delay. |
| Cursor sticking on touch taps | Medium | Query touch capabilities on mount and disable the custom elements layout on touch viewports. |

## Tasks

### Task 1: Build Custom Trailing Cursor Component

Create the cursor dot and trailing outer circle wrapper synced to mouse movements.

<read_first>
- `components/ui/CustomCursor.tsx` [NEW]
- `app/layout.tsx`
- `app/globals.css`
</read_first>

<acceptance_criteria>
- Custom cursor accurately tracks mouse pointer position.
- Trailing ring moves with a soft physics delay using GSAP springs.
- CSS `cursor: none` applied on desktop.
</acceptance_criteria>

<action>
1. Create `components/ui/CustomCursor.tsx` as a client component:
   - Use `gsap.quickTo` for `x` and `y` positions of the outer ring wrapper.
   - Bind mousemove event listeners to update cursor coordinates.
   - Clean up event listeners on unmount.
2. In `app/globals.css`, apply `cursor: none` globally to HTML/Body elements on screen sizes `min-width: 768px`.
3. Wrap `app/layout.tsx` children in the `<CustomCursor />` component.
</action>

---

### Task 2: Implement Dynamic Hover States & Labels

Configure hover scaling and text insertions when interacting with links, buttons, and card containers.

<read_first>
- `components/ui/CustomCursor.tsx`
- `app/globals.css`
</read_first>

<acceptance_criteria>
- Hovering over buttons/links scales up the cursor.
- Hovering over elements with `data-cursor="view"` scales the ring and displays "View" text inside it.
</acceptance_criteria>

<action>
1. Update `components/ui/CustomCursor.tsx`:
   - Query all elements with `hoverable` or tags `a`, `button`, `[data-cursor]`.
   - Add mouseenter/mouseleave bindings to update cursor state ref (scale ring, set text content inside ring).
   - Use dynamic data attributes like `data-cursor="view"` to set the inner text.
</action>

### Task 4: Prevent Mobile/Touch Cursor Bugs

Ensure the custom cursor disappears on touch screens.

<read_first>
- `components/ui/CustomCursor.tsx`
</read_first>

<acceptance_criteria>
- Touch viewports do not render the custom cursor and preserve standard browser touch feedback.
</acceptance_criteria>

<action>
1. In `components/ui/CustomCursor.tsx`, check `navigator.maxTouchPoints > 0` or standard touch media query matches.
2. If true, return `null` (do not render elements).
</action>
